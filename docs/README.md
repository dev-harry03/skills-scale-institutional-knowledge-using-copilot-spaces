# OctoAcme Project Management Documentation

Welcome to OctoAcme's centralized project management knowledge base. This documentation captures the processes, workflows, and best practices we use to deliver projects successfully across the organization.

## Overview

OctoAcme follows an **iterative, stakeholder-aligned approach** to project management with clearly defined phases and responsibilities. Our methodology prioritizes:

- **Customer-first thinking**: prioritize customer value and usability
- **Iterative delivery**: deliver small, testable increments with regular feedback loops
- **Clear ownership**: each project has defined Project Manager and Product Lead roles
- **Data-informed decisions**: measure impact and iterate based on evidence
- **Psychological safety**: encourage feedback, learning, and continuous improvement

## Project Lifecycle

OctoAcme projects follow a structured lifecycle:

1. **Initiation**: Define the problem, identify stakeholders, establish high-level timeline
2. **Planning**: Break work into shippable increments, identify risks and dependencies
3. **Execution**: Build, test, review, and iterate with regular team rhythms
4. **Release**: Deploy to production, verify functionality, and announce to stakeholders
5. **Retrospective**: Capture learnings and convert into actionable improvements

## Core Roles

| Role | Responsibility |
|------|-----------------|
| **Project Manager (PM)** | Coordinates delivery, manages schedules, risks, and communications |
| **Product Manager (PdM)** | Defines outcomes, prioritizes backlog, and measures success |
| **Developers** | Implement features, collaborate on design, and ensure code quality |
| **QA/Testing** | Validate quality and acceptance criteria |
| **Stakeholders** | Provide inputs, approvals, and strategic guidance |

For detailed role descriptions, see [Roles & Personas](octoacme-roles-and-personas.md).

## Key Artifacts

Every OctoAcme project maintains:

- **Project Charter / One-pager**: Problem statement, goals, success metrics, stakeholders
- **Roadmap & Release Plan**: Timeline, milestones, release schedule
- **Sprint/Iteration Backlog**: Prioritized work with acceptance criteria
- **Risk Register**: Identified risks with impact, likelihood, and mitigation plans
- **Retrospective Notes**: Learnings and action items for continuous improvement

## Communication Cadence

- **Daily Standups**: 15-minute team syncs focused on progress, blockers, and dependencies
- **Weekly PM Sync**: Product Manager and Project Manager alignment
- **Weekly Stakeholder Updates**: Status, risks, blockers, and decisions needed
- **Sprint Reviews/Demos**: End-of-sprint showcase of completed work
- **Sprint Retrospectives**: Team reflection and process improvements
